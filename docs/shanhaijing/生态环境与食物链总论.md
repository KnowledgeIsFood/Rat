# 生态环境与食物链总论（设计推演）

经文本身 **很少给出完整现代意义上的“生态系统”**，但反复出现可抽象的生态要素：**山川走向、水脉注海、林相（多松、多梓、无草木）、矿物色（丹雘、瑶碧）、祥/凶异兆（见则大水、见则大旱）**。本资料层在每条目中用两块文字承接：

1. `habitat_ecology_zh`：把 **地貌 + 水文 + 林相 + 矿物色** 组合成可关卡化的环境标签（湿地、裸岩、焚风谷、石漠等）。
2. `food_web_zh`：在 **不违背经文边界** 的前提下，用现代生态学语言做 **弱耦合推演**（谁吃谁、谁是顶级消费者、是否存在“灾害位”改变环境）。

## 可复用的几条设计规则

- **食人兽簇**：彘、蠱雕、九尾狐、狍鸮、诸怀、穷奇、巴蛇等，适合共用“高危区”生态位与 AI 模板，但要用 **地貌 / 水系 / 声纹** 做差异化。
- **水患链**：长右、夫诸、化蛇等“见则大水”单位，可与关卡 **潮汐 / 涨落 / 河道改线** 机制绑定。
- **旱灾链**：肥遗（蛇形）、顒、鱄鱼等“见则大旱”单位，可与 **枯萎、火风、资源枯竭** 绑定。
- **祥瑞 / 神祇位**：凤凰、鸾鸟、乘黄、帝江、陆吾、开明兽等，尽量 **弱耦合真实食物链**，改以 **仪式、结界、权能** 叙事供能。

## 与“异变”玩法的接口

异变不仅是数值成长，可把 **生态位反转** 作为母题：草食异变出角刺、滤食鱼异变为底栖暴君、祥瑞黑化为疫土载体等——均应在条目中留下 `traits_effects_zh` 与 `food_web_zh` 的伏笔句，便于系统策划回溯。
