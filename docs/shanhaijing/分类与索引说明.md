# 分类与索引说明

## 一级：类目 `kind_zh`

用于筛选 **奇禽 / 异兽 / 鳞介 / 蛇蟒 / 草木 / 木本 / 神祇 / 祥瑞 / 凶兽** 等；与 `chapter_group`（写哪份 JSON）正交。

## 二级：典籍块（corpus / jing）

机读字段：

- `corpus_zh`：如「五藏山經」「海經」「大荒經」。
- `jing_zh`：如「南山經」「海内西經」「大荒北經」。
- `section_zh`：更细的篇次（如「南次二經」「西次三經」）。

## 二级：形态标签（建议后续在 JSON 增加 `tags[]`）

便于肉鸽构筑筛选，可逐步加入，例如：

- `兽`、`鸟`、`鱼`、`蛇`、`龟`、`虫`、`复合`、`神祇`、`祥瑞`、`凶兆`、`灾害`、`食人`。

当前 v1 未单列 `tags`，可由策划在 `emit` 脚本中追加。

## 三级：稳定 ID（`id`）

- 使用 **小写 ASCII slug**，与存档、本地化键、Godot 资源名兼容。
- **同名异物** 必须分 id（例：`fei_snake` 与将来可增补的 `fei_wing` 六足四翼肥遗）。

## 人类可读索引

- 总表：`docs/shanhaijing/CATALOG.md`（脚本生成，勿手改）
- 分条：`docs/shanhaijing/creatures/by_id/<id>.md`（脚本生成，勿手改）
- 源本：`data/shanhaijing/chapters/*.json`（由 `tools/emit_shanhaijing_chapters.py` 生成）
