# 异兽主表（CSV）维护说明

## 文件位置

- **主表（请在此追加行）**：`data/shanhaijing/creatures_master.csv`
- UTF-8 **带 BOM**（`utf-8-sig`），便于 Excel 直接打开且不乱码。

## 列说明

| 列名 | 必填 | 说明 |
|------|------|------|
| `chapter_group` | 是 | 输出到哪个 JSON：`nanshan_jing` / `xishan_jing` / `beishan_jing` / `dongshan_jing` / `zhongshan_jing` / `haijing` / `dahuang_jing`（与 `data/shanhaijing/chapters/<名>.json` 对应） |
| `id` | 是 | 全表唯一 ASCII slug，建议小写+下划线 |
| `name_zh` | 是 | 中文正名 |
| `aliases_zh` | 否 | 多个别名用 **竖线 `\|`** 分隔，如 `别名甲\|别名乙` |
| `corpus_zh` | 是 | 大部头，如 `五藏山經` / `海經` / `大荒經` |
| `jing_zh` | 是 | 篇名，如 `南山經`、`海内西經` |
| `section_zh` | 是 | 更细篇次或列系说明 |
| `mountain_zh` | 是 | 山、水、泽等所在 |
| `literature_appearance_zh` | 是 | 文述画像（形体大意） |
| `traits_effects_zh` | 是 | 特点与异效（含玩法映射） |
| `habitat_ecology_zh` | 是 | 生态环境（推演） |
| `food_web_zh` | 是 | 生物链与种间关系（推演） |
| `trophic_role_zh` | 是 | 营养级/设计标签 |
| `artistic_portrait_note_zh` | 是 | 美术转化要点 |
| `related_zh` | 否 | 关联条目标题或关键词，多个用 **`\|`** 分隔 |
| `sources_note_zh` | 否 | 资料来源与校勘说明 |

## 单元格里的逗号、换行、引号

- 文本中可照常使用中文逗号 `，`。
- 若使用 **英文逗号** 或 **换行**，建议用 Excel/LibreOffice 编辑保存为 CSV，编辑器会自动加引号；或用纯文本编辑器并整体用双引号包裹该单元格。
- 英文双引号在单元格内需写成 **`""`**（标准 CSV 转义）。

## 生成下游文件

在仓库根目录执行：

```bash
python3 tools/emit_shanhaijing_chapters.py
python3 tools/render_shanhaijing_docs.py
```

会更新：

- `data/shanhaijing/chapters/*.json`
- `docs/shanhaijing/CATALOG.md` 与 `docs/shanhaijing/creatures/by_id/*.md`
- `game/data/creatures_catalog.json`

**不要手改**上述生成物，一律改 CSV 后重跑脚本。

## `chapter_group` 与新增分组

默认会写出 `emit` 脚本里列出的 7 个分组文件。若在 CSV 中使用新的 `chapter_group` 名，脚本也会额外生成 `data/shanhaijing/chapters/<新名>.json`；同时请记得让 `tools/render_shanhaijing_docs.py` 能读到（已支持 `chapters/*.json` 通配），并在版本控制中纳入新文件。
