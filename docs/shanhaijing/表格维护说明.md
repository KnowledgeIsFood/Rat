# 异兽 / 草木主表（CSV）维护说明

## 文件位置

- **主表（请在此追加行）**：`data/shanhaijing/creatures_master.csv`
- UTF-8 **带 BOM**（`utf-8-sig`），便于 Excel 直接打开且不乱码。
- **批量扩表（可选）**：`python3 tools/expand_shanhaijing_master_csv.py` 会重写主表：为旧行补 `kind_zh`，并追加程序化占位行（见下文「占位」）。

## 列说明

| 列名 | 必填 | 说明 |
|------|------|------|
| `chapter_group` | 是 | 输出到哪个 JSON，与 `chapters/<名>.json` 对应。**游戏主线为五藏山经**：动物分组 `nanshan_jing` / `xishan_jing` / `beishan_jing` / `dongshan_jing` / `zhongshan_jing`；草木分组 `plants_nanshan_jing` … `plants_zhongshan_jing` 及 `plants_wucang_misc`。**`haijing` / `dahuang_jing` 为扩展**，非五篇山经主轴，见 `游戏主题-五藏山经.md`。 |
| `kind_zh` | 是 | **类目**：如 `奇禽`、`异兽`、`鳞介`、`蛇蟒`、`草木`、`木本`、`神祇`、`祥瑞`、`凶兽` 等；草木类多用 `草木` / `木本`。 |
| `id` | 是 | 全表唯一 ASCII slug，建议小写+下划线 |
| `name_zh` | 是 | 中文正名 |
| `aliases_zh` | 否 | 多个别名用 **竖线 `\|`** 分隔 |
| `corpus_zh` | 是 | 大部头，如 `五藏山經` / `海經` / `大荒經` |
| `jing_zh` | 是 | 篇名，如 `南山經`、`海内西經` |
| `section_zh` | 是 | 更细篇次或列系说明 |
| `mountain_zh` | 是 | 山、水、泽等所在 |
| `literature_appearance_zh` | 是 | 文述画像（形体 / 植株大意） |
| `traits_effects_zh` | 是 | 特点与异效（含玩法映射） |
| `habitat_ecology_zh` | 是 | 生态环境（推演） |
| `food_web_zh` | 是 | 种间关系：动物写捕食链；草木可写「被植食兽取食 / 昆虫访花 / 人采」等 |
| `trophic_role_zh` | 是 | 营养级或设计标签（草木常用 `初级生产者`） |
| `artistic_portrait_note_zh` | 是 | 美术转化要点 |
| `related_zh` | 否 | 多个用 **`\|`** 分隔 |
| `sources_note_zh` | 否 | 资料来源与校勘说明 |

## 关于「补全」与占位行

《山海经》全书异兽草木条目数量远大于当前仓库收录量。`expand_shanhaijing_master_csv.py` 会追加两类占位：

1. **未具名异兽占位**：`synth_<chapter_group>_01` … 等，用于先铺满关卡密度，后续请替换为注本可核的专名。
2. **草木归纳占位**：`plant_<chapter_group>_..` 等，为常见科属归纳，**非**逐条经文校勘。

若你只做精校条目，可删除占位行后保留主表，再跑 `emit` / `render`。

## 单元格里的逗号、换行、引号

- 文本中可照常使用中文逗号 `，`。
- 若使用 **英文逗号** 或 **换行**，建议用 Excel/LibreOffice 编辑保存为 CSV；英文双引号在单元格内需写成 **`""`**。

## 生成下游文件

```bash
python3 tools/emit_shanhaijing_chapters.py
python3 tools/render_shanhaijing_docs.py
```

会更新：`data/shanhaijing/chapters/*.json`、`docs/shanhaijing/CATALOG.md`、`docs/shanhaijing/creatures/by_id/*.md`、`game/data/creatures_catalog.json`。

**不要手改**上述生成物，一律改 CSV 后重跑脚本。

## 新增 `chapter_group`

若在 CSV 中使用新的 `chapter_group` 名，`emit` 会额外写出对应 `chapters/<新名>.json`；`render` 已通配合并所有 `*.json`。
